Document Purpose

This document summarizes the changes in each release.

Intended Audience

This document may be of interest to anyone who installs or uses the application.

Version 1.0.0.552

New Features

This was the initial release. It contained the following features:

  • has an integrated installer
  • runs in the background as a low-priority Windows NT service
  • uploads files to an SFTP server
  • downloads configuration file from an SFTP server
  • uploads system information to an SFTP server
  • scans filesystem for a list of files to be uploaded
  • runs jobs on a schedule
  • automatically upgrades itself when a new version is available

Enhancements

None

None

Known Issues:

None

Version 2.0.0.1934

New Features

The following new features were added:

  • security information can be downloaded from an HTTPS server
  • security information can be entered manually if necessary
  • strips non-printable and whitespace characters from ODBC extracts and SQL statements
  • ODBC extracts will now contain a header on line 1 that lists the field names
  • you can now specify which character to use as a field delimiter in ODBC extracts
  • files.xml now prints each file's datestamp in human-readable format
  • odbc.xml can contain a description for each resource
  • you can create variables in security.xml and reference them from odbc.xml in the SQL or the filename
  • a built-in variable called #DateStamp# has been added that will expand to the current date/time using this format: yyyy.MM.dd.HH.mm.ss
  • more icons have been added to the Start menu

Enhancements

The following changes were made to existing features:

  • download size reduced from 66MB to 50MB
  • GUI utility for configuration management removed
  • installer prompts for security information
  • added new screens to the installer for managing security information
  • user may enter security information manually in installer if automatic method fails
  • split configuration file into 3 separate files
  • migrated Windows registry reads/writes from Perl to Java
  • Java launchers are now built using a new utility
  • splash screens are now displayed by native launchers instead of HTA files
  • installer now works with Windows Installer
  • upgraded to newer JDK
  • added product version number to sysinfo.xml and server.log

Defects Resolved:

None

Known Issues:

  • The new installer will prompt you for an installation directory, but do not change it. You must use the default value. This may be corrected in a future release.
